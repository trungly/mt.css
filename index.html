Minithematiq.css

Intro

Welcome to Minithematiq ("MT"), a lightweight CSS starter evolved from my personal projects. I grab MT whenever I need to hack something quick. Currently I'm using it for Kibble.js, and its first published site MattMade.me. Originally intended as a "super minimalist" CSS framework, MT has now grown a bit beyond that, but.. it sure ain't Bootstrap! Then again, sometimes "less is more" :)

  • Made with Sass: It's the good stuff
  • Modular: Include only what you need
  • Responsive Grid: Works across desktops & mobile
  • Theme Vars File: Quick to mod appearance
  • Resizable: Majority of sizing & spacing derived from one base unit
  • Prefixed: Unlikely to clash with your other styling

Since it's not an "official" project, the docs here are pretty sparse, but you can always do the ol' Inspect Element for more detail. If anyone finds any value in MT (or even finds it here to begin with), hey, awesome!

Getting Started

Option A: Grab & Run

Just download the css/mt.css file from the repo and include it in your page. I'd also recommend that you first reset via normalize.css, as I do here. Of course, this method will not allow you to make any mods.

Option B: Compile Away

Grab all of the .scss files from _dev/sass/ and compile your own, either using the Ruby Sass command line tool, CodeKit, Grunt, Gulp, etc.

Alternatively, assuming you have NPM already installed, you can git clone https://github.com/pmdesgn/mt.css.git this entire repo, then npm install will net you my Gulp build setup with LiveReload-enabled local server and all. Just run gulp to build and serve (on http://localhost:8080). To compile a minified version for prod, do gulp --type prod. (If you haven't used Gulp before, you may need to install it "globally" first via npm install -g gulp, so your system knows where to find it.)

The mt.scss is the entry-point file which includes all other MT module imports. You don't need all of the partials; say, you may only want mt-theme-vars.scss and mt-grid.scss. Just remove (or comment out) what you don't need. The minimum requirement is mt-theme-vars.scss -- if you want to start modifying MT, this is the file to edit first.

Wrappers

mt-wrapper mt-row

Use mt-wrapper to keep your inner content within a centered, width-limited container, as defined in the var file.

Use mt-rowto wrap your grid columns, or anytime you want a block container with a bit of vertical margin.

Fluid Grid

Columns

When specifying column widths, MT foregoes "semantics" in favor of referencing percentage widths directly. What can I say, personal preference. The following column classes are supported, with the narrowest being 20%, thus allowing up to five columns per row:

mt-20 mt-25 mt-33 mt-40 mt-50 mt-60 mt-66 mt-75 mt-80

These columns are floated left. The class mt-100 also applies the block property.

Remember to wrap any row of columns inside a div class of mt-row for the grid to "behave" (floats cleared).

Responsive: Resolution Thresholds

MT ships with three basic resolution thresholds, cascading down in this order:
mt-max*Above 720px>
mt-med481 - 720px>
mt-minUp to 480px
By controlling your column widths, offsets, and visibility at each resolution, you can achieve most common fluid layouts. The base column classes apply across all resolutions by default, starting with mt-max, so if you want responive behaviors, you only need to define what happens at mt-med and/or mt-min.

Responsive: Columns & Offsets

For example, you can start your column div at 50% width, then switch to 100% at the mt-med threshold. Resize your browser window to see it in action:
mt-50  mt-med-100
mt-50  mt-med-100
You can specify more than one threshold action. Say, starting at 1/3 wide, switching to 1/2 at mt-med, and finishing at 100% at mt-min:
mt-33  mt-med-50  mt-min-100
Column offsets can be controlled in the same fashion. Say, an initial tab of 33%, reduced to 20% at mt-med, then zero via mt-min-100:
mt-tab-33  mt-33  mt-med-tab-20  mt-med-60  mt-min-100

Responsive: Visibility

Start element as visible, then hide it when a specific threshold is reached. Resize your browser window to see it in action:
mt-max-hideOla!
mt-med-hideOla!
mt-min-hideOla!
Start element as hidden, then show it when a specific threshold is reached:
mt-max-showOla!
mt-med-showOla!
mt-min-showOla!

Helpers

These "shortcut" classes would likely give the "semantics police" a heart attack, but as I hinted above I'm not super OCD about it, especially when trying to style a bit of HTML quickly. Some examples, hopefully self-explanatory:
mt-left mt-clear mt-block mt-text-center mt-no-padding mt-margin-top mt-color-fg mt-bg-color-accent
For the full list, see the actual _mt-helpers.scss partial.

These two convenient palette "shortcuts" reside in the main MT file, so they are not dependent on the Helpers partial:

mt-palette-flip inverse-like, flips the fg and bg colors.

mt-palette-accent applies the accent color to the background and bg to the foreground.
